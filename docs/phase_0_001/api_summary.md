# Phase 0.001 API Summary

AeroCodex Phase 0.001 provides source-traceability scaffolding and first-pass checked scalar aerospace equations. All equation metadata remains conservative until source review upgrades it.

- Core: `AeroResult<T>`, `AeroError`, `EngineeringResult<T>`, `Assumption`, `ModelWarning`, `ValidityStatus`, `VerificationStatus`, and `VerificationRecord`. Status types display and parse as canonical snake-case strings without adding serialization dependencies.
- Unit scalars: `Angle`, `Mach`, `Gamma`, `Pressure`, `Temperature`, `Density`, `Length`, `Area`, `Mass`, `Time`, `Velocity`, `Acceleration`, `Force`, and `HeatFlux`. Nonnegative physical magnitudes use checked constructors; `Force` and `HeatFlux` remain signed with checked finite constructors for untrusted input.
- Constants: `aero-codex-constants` exports Phase 0.001 seed constants for standard gravity, gas constants, sea-level atmosphere values, dry-air gamma, Stefan-Boltzmann, Earth astrodynamics values, and an explicitly unverified solar gravitational-parameter placeholder. `PHASE_0_001_CONSTANT_SEEDS` and `constant_seed(symbol)` expose dependency-free metadata, with every source status held at `research_required`.
- Validation scaffold: `validation/schema/codex_card.schema.json` defines the Codex Card structure; `validation/cards/` contains planning cards; `validation/source_registry/` contains source-research seeds; `xtask` can check schema markers, cards, source-registry links, statuses, categories, nonempty list sections, and dependency policy without adding parser dependencies.
- Atmosphere: `standard_sea_level`, `troposphere_temperature`, `troposphere_pressure`, `troposphere_density`, `troposphere_state`, `speed_of_sound`, and `verification_record`. The Phase 0.001 troposphere model accepts `0 m <= altitude_m <= 11000 m`, documents the geometric/geopotential altitude simplification, and keeps trace metadata at `research_required`.
- Thermodynamics: `ideal_gas_density`, `speed_of_sound`, `cp_from_gamma_r`, `cv_from_gamma_r`, `gamma_from_cp_cv`, `specific_gas_constant_from_molar_mass`, and `verification_record`. The Phase 0.001 perfect-gas helpers require positive absolute temperature for density and speed of sound, keep all trace metadata at `research_required`, and avoid CEA/native/property-database dependencies.
- Gas dynamics: reviewed direct isentropic helpers `temperature_ratio_t0_over_t`, `pressure_ratio_p0_over_p`, `density_ratio_rho0_over_rho`, `area_mach_ratio`, and `mass_flow_parameter`; reviewed direct normal-shock helpers `normal_shock_mach2`, `normal_shock_pressure_ratio_p2_p1`, `normal_shock_density_ratio_rho2_rho1`, `normal_shock_temperature_ratio_t2_t1`, and `normal_shock_total_pressure_ratio_p02_p01`; reviewed expansion-flow helpers `mach_angle`, `prandtl_meyer_nu`, and `prandtl_meyer_inverse`; reviewed branch-explicit oblique-shock helpers `theta_beta_mach_residual`, `oblique_shock_beta`, `oblique_shock_normal_mach`, and `oblique_shock_downstream_mach`; and provides conservative `verification_record` metadata for reviewed Codex IDs. Reviewed outputs and inverse solves are checked for finite values or numerical failure, weak/strong oblique-shock branch selection is explicit, and all gas-dynamics trace metadata remains `research_required`.
- Aerodynamics: reviewed `dynamic_pressure`, `lift`, `drag`, `lift_coefficient`, `drag_coefficient`, `induced_drag_coefficient`, and `verification_record`. Dynamic pressure rejects negative density and speed, force helpers reject negative reference area, coefficient inverses require positive `q` and area, induced drag requires positive aspect ratio and Oswald efficiency, and all trace metadata remains `research_required`.
- Propulsion: reviewed `tsiolkovsky_delta_v`, `mass_ratio_from_delta_v`, `ideal_thrust`, `specific_impulse_from_effective_exhaust_velocity`, `choked_mass_flux_per_area`, and `verification_record`. The Phase 0.001 helpers keep rocket/nozzle assumptions idealized, preserve signed pressure-thrust bookkeeping, guard nonfinite derived outputs, and keep all propulsion trace metadata at `research_required`.
- Heat transfer: reviewed `stefan_boltzmann_radiative_flux`, `convective_heat_flux`, `thermal_resistance_conduction`, `conduction_heat_rate`, and `verification_record`. The Phase 0.001 helpers preserve signed heat-flux/heat-rate conventions where temperature ordering is caller supplied, guard nonfinite derived outputs, and keep all heat-transfer trace metadata at `research_required`.
- Structures: reviewed `axial_stress`, `bending_stress`, `cantilever_tip_deflection_end_load`, `euler_column_buckling_load`, and `verification_record`. The Phase 0.001 helpers preserve signed force/moment/section-coordinate conventions where supplied, require positive area/length/elastic-modulus/second-moment/effective-length-factor domains as applicable, guard nonfinite derived outputs, and keep all structures trace metadata at `research_required`.
- Flight dynamics: reviewed `load_factor_level_turn`, `turn_rate`, `turn_radius`, `stall_speed`, `specific_excess_power`, and `verification_record`. The Phase 0.001 helpers document steady level coordinated-turn assumptions, reject singular bank-angle/invalid performance domains, guard nonfinite derived outputs, preserve signed turn-rate and specific-excess-power conventions where supplied, and keep all flight-dynamics trace metadata at `research_required`.
- Astrodynamics: reviewed `circular_orbit_speed`, `orbital_period_circular`, `escape_velocity`, `vis_viva_speed`, `specific_orbital_energy`, `hohmann_transfer_delta_v1`, `hohmann_transfer_delta_v2`, `hohmann_transfer_total_delta_v`, `hohmann_transfer_time`, `sphere_of_influence_radius`, and `verification_record`. The reviewed helpers require positive `mu`, positive radii, and positive semi-major axis/mass/distance where applicable; reject negative vis-viva radicands; guard nonfinite derived outputs; preserve Hohmann burn magnitudes for either transfer direction; and keep trace metadata at `research_required`.
- Life support: reviewed `closure_fraction`, `required_production_area`, `buffer_residence_time`, `crew_daily_requirement`, `net_daily_balance`, optional `oxygen_balance`, `carbon_dioxide_balance`, `water_recovery_balance`, and `verification_record`. The Phase 0.001 helpers are scalar mass-balance primitives only; closure-style values above one carry warning and `OutsideDocumentedDomain` metadata, nonfinite derived outputs are guarded, and all life-support trace metadata remains `research_required`.

Each equation crate uses `AeroResult<T>` where invalid input is possible. The life-support closure and water-recovery functions return `EngineeringResult<f64>` so warnings, assumptions, validity, and conservative verification metadata can travel with the scalar value.
